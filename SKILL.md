# Tanue Technical Narrative Writing Skill

## 适用文章类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 技术史叙事 | 航空航天、军事装备、工业史、汽车史、核能技术 | V-2火箭、图-64、M-17、拉达汽车 |
| 工程事故复盘 | 航空/航天/核事故系统性因果链分析 | 切尔诺贝利、737MAX、SSJ100 |
| 人物致敬 | 科学家、宇航员、工程师 | 列昂诺夫 |
| 个人情绪叙事 | 异地、车站、远方、青春、离别意象 | 月影车站 |

## 核心写作原则

### 1. 不要百科式开头

❌ V-2 火箭是德国在二战期间研制的弹道导弹……
✅ 时间拨回到上世纪三十年代。彼时的人类还没有真正离开地球，但已经有人望着星空，认真地思考如何把一个金属圆筒送到更高的地方。

### 2. 参数必须服务于解释

每个数字都要说明它意味着什么、解决了什么问题、带来了什么新问题。

❌ A-3 火箭高度 6.5 米，直径 75 厘米，发动机推力约 15000 牛。
✅ A-3 火箭高度约 6.5 米，直径 75 厘米，发动机推力约 15000 牛。这个尺度已经不再是早期火箭爱好者的小型试验品，而是开始逼近真正武器系统的雏形。

### 3. 事故用多因素叠加

不要简单归因。写成因果链：表面原因 → 直接技术机制 → 操作因素 → 组织因素 → 历史/制度背景 → 系统性教训

### 4. 区分事实/推断/观点

- **事实**：直接陈述，给时间地点型号
- **推断**：标明不确定性（"从现有资料看……很可能……"）
- **观点**：有依据地表达（"与其说……不如说……"）

## 模式A：改写已有草稿

**适用**：用户提供草稿，要求改成 Tanue 风格。

**三步流程**：

**第一步：开头改造。** 将百科式开头改成从历史场景、事故瞬间、反常问题切入。

**第二步：段落结构调整。** 打破每段150-250字的均匀结构。断出微段（<30字），图片说明单独成行，删除段尾升华句。

**第三步：语言质感改造（按优先级）。**
- P1：去除非必要引号（去70%以上），只保留正式名称、直接引语、项目代号
- P2：替换禁止句式（全文合计≤3次）："不是……而是……""这说明……""这标志着……""从某种意义上说……"
- P3：替换抽象词：命运→经历，转身→改成，遗产→留下，叙事→记录，高光→公开亮相
- P4：参数附意义，每个数字都要有解释
- P5：事故因果链化，写多因素叠加
- P6：图片说明简短化（≤35字，单独成行）

**不改**：事实、结构、图片、格式。

## 模式B：从头创作

**适用**：用户只说"帮我写一篇关于XX的文章"。

### 四种文章模板

#### 模板A：技术史长文

**标题**："对象 + 历史定位/反差/问题"
**开头**：写作动机 + 历史入口
**主体**：时代背景 → 时间线发展 → 技术特点（边介绍边解释） → 成败分析（至少三层）
**结尾**：从对象上升到时代

#### 模板B：工程事故复盘

**标题**：设问式或反差式
**开头**：提出反常问题
**主体**：事故概况 → 完整时间线（按时间推进） → 技术机制解释 → 多因素叠加分析 → 工程教训
**结尾**：克制收束

#### 模板C：人物致敬

**开头**：死亡/纪念/历史节点切入。不写生平流水账。
**主体**：人物 + 任务 + 险情 + 历史意义
**结尾**：庄重不轻佻，可引用经典

#### 模板D：个人情绪叙事

**标题**：有意象（车站、月光、雨、列车）
**视角**：第三人称切入，最后转第一人称
**情绪**：克制、延迟、靠意象累积

## 输出风格三档

| 档位 | 特点 | 适用 |
|------|------|------|
| 知乎长文风 | 口语转场多、作者存在感强、可有"笔者注" | 知乎、公众号初稿 |
| 成熟科普风 | 结构清晰、语言克制、技术解释准确 | 公众号正式稿、个人品牌 |
| 准学术叙事风 | 保留叙事入口、资料严谨、减少情绪 | 讲稿、报告、课程材料 |

## 常用转场句

- 接上文。
- 现在我们回到正题。
- 时间拨回到……
- 这就产生了一个问题。
- 事情到这里还没有结束。
- 值得一提的是……
- 当然，这是后话。
- 看似简单，实际并不简单。
- 但问题恰恰出在这里。

## 检查清单

- [ ] 开头是从场景/事故/问题切入（非百科定义）？
- [ ] 回答了"为什么值得写"？
- [ ] 参数附带了意义说明？
- [ ] 禁止句式合计不超过3次？
- [ ] 段落长短不均匀？
- [ ] 引号控制在必要范围？
- [ ] 抽象词替换为具体说法？
- [ ] 事故有因果链分析？
- [ ] 区分了事实/推断/观点？
- [ ] 结尾不过度煽情？

---

## 一句话概括

> 先让读者进入一个场景，再让读者理解一套机制，最后让读者记住一个判断。
